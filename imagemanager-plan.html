Codebase Diagnosis — What's Actually Broken

DeepFace dual-pipeline mismatch critical
SimplifiedFaceService calls DeepFace.represent() for embedding and relies on the old FaceDetectionService which calls extract_faces() separately. The two preprocessing paths produce inconsistent embeddings for the same face — this is the primary cause of person fragmentation (same person → multiple IDs).
Threshold calibration mismatch critical
Config sets similarity_threshold: 0.50 but IdentityAssignmentService defaults to 0.80. The EnhancedIdentityAssignmentService calls embedding_service.cosine_similarity(), which is not defined on EmbeddingService — it exists only on SimplifiedFaceService. This causes an AttributeError at runtime when Stage 4 is enabled.
pickle embedding serialization critical
All 512-dim vectors are stored with Python's pickle into LargeBinary columns. This is version-fragile, slow for bulk loads, and makes vector similarity search impossible in SQL. A full table scan + deserialize happens for every new face processed — O(n·F) where F = faces per person.
FAISS wired but unusable critical
FAISSVectorStore is built and integrated in main.py but search_by_person() references an undefined Dict type (missing import). The FAISS IVF index also requires training on ≥ nlist vectors before any search — this is not handled, causing a silent crash on cold start.
DeepFace is a heavyweight wrong tool architecture
DeepFace downloads and wraps TensorFlow/Keras models at ~600 MB+ (ArcFace weights). It adds TF overhead, has no ONNX path, and mixes detection + embedding in an opaque pipeline. InsightFace with its ONNX buffalo_l pack is faster, smaller, more accurate, and gives direct control over each stage.
Auto-merge runs in-session mid-pipeline logic
detect_and_merge_duplicates() is called inside the same DB session as ingestion. If a merge commits and fails mid-way, partial merges leave orphaned ClusterDB rows. The image→person remapping after a merge uses a raw SQLAlchemy query with a missing model alias (cluster_repo.Cluster doesn't exist as a property).
PersonRepository.get_by_name missing runtime error
scripts/rename_person.py and merge_persons.py both call person_repo.get_by_name() but this method is not defined in PersonRepository — only get_by_id() exists. All rename and manual merge scripts will crash immediately.
Quality score formula is naive accuracy
SimplifiedFaceService computes quality as min(1.0, face_area / 10000) — a 100×100 pixel face scores 1.0. This produces uniform high-quality scores for most faces and makes quality-weighted selection in Stage 4 useless. Blur detection (Laplacian variance) in FaceDetectionService is never called by the simplified path.
FaceRepository missing get_by_image missing method
main.py calls face_repo.get_by_image(img_record.image_id) in the auto-merge remapping block — this method does not exist in FaceRepository. Dead code path that errors silently only when auto-merge fires.
Config get() returns wrong types subtle
YAML parses numeric port as int, but config.get() returns raw values without coercion. When env-var override sets DB_PORT as string, SQLAlchemy connection fails with a type error. The ClusteringService receives similarity_threshold=0.70 from config but uses 0.85 internally in merge_clusters().

  • Fix FAISSVectorStore.search_by_person(): add from typing import Dict

  • Unify similarity_threshold: remove all hardcoded defaults, always read from config

Config Fixes

  • Remove auto_merge_duplicates: true from default config (too risky as default)
  • Add ClusteringService tests to verify DBSCAN output
  • Add migration_v2.sql: run without breaking existing data
PHASE 2 Replace DeepFace with InsightFace (ONNX) ~2–3 days

New Service: insight_face_service.py

  • Install: pip install insightface onnxruntime

  • Single process_image(path) call returns both detection + embedding
  • Expose: embedding, det_score, bbox, kps (landmarks)
  • Quality: det_score × laplacian_clipped × size_ratio — real three-factor score
  • Implement calculate_similarity(e1, e2) using np.dot (L2-normalized)

Deprecate

  • Remove services/face_detection_service.py
  • Remove services/simplified_face_service.py
  • Remove services/embedding_service.py
  • Remove services/http_face_service.py
  • Remove ai_service/ directory entirely

PHASE 3 Migrate to pgvector for Vector Storage & Search ~2–3 days

Database Changes

  • Use Docker image pgvector/pgvector:pg16
  • Run CREATE EXTENSION IF NOT EXISTS vector; in init script
  • Add pip install pgvector
  • Change FaceDB.embedding: LargeBinary → Vector(512)
  • Change ClusterDB.center_embedding: LargeBinary → Vector(512)
  • Migration script: deserialize existing pickles → insert as vectors
  • Add CREATE INDEX ON faces USING ivfflat (embedding vector_cosine_ops) WITH (lists=100);

Repository Changes

  • Remove all pickle.dumps/loads from repositories
  • Add FaceRepository.find_nearest(embedding, k, threshold) using <=>
  • Return (person_id, cluster_id, distance) from ANN search
  • Remove services/faiss_vector_store.py
  • Remove services/faiss_identity_assignment_service.py
  • Update ClusterRepository.update_center(): pass np.ndarray directly
PHASE 4 Consolidate Identity Assignment ~1–2 days

New IdentityService

  • Single class replaces all three: IdentityAssignmentService, EnhancedIdentityAssignmentService, FAISSIdentityAssignmentService
  • Use pgvector ANN for candidate retrieval (fast)
  • Then verify top-K candidates against stored face embeddings (accurate)
  • One threshold value injected from config, nowhere else
  • All DB writes in a single session.flush() per face — no partial commits
  • Return AssignmentResult(person_id, is_new, similarity, cluster_id) dataclass

Cluster Update Logic

  • Keep one cluster per person by default (simpler, better)
  • Incremental center: new_center = (old_center × n + new_emb) / (n+1)
  • No full re-scan on every new face
  • Recompute center from scratch only on merge_persons
  • Quality-weighted top-5 comparisons for multi-embedding match before ANN confirm
PHASE 5 Fix Scripts, CLI & Operational Tooling ~1 day

All Scripts

  • scripts/rename_person.py — add PersonRepository.get_by_name(), fix crash
  • scripts/merge_persons.py — run in isolated transaction, fix get_by_name
  • scripts/detect_duplicates.py — use pgvector ANN instead of O(n²) loop
  • scripts/auto_merge.py — extract to standalone script with its own session, never run mid-pipeline
  • scripts/person_stats.py — works as-is after repo fixes
  • scripts/reset_db.py — add TRUNCATE ... RESTART IDENTITY CASCADE
  • Add scripts/export_faces.py: export face crops per person for visual audit
  • Add scripts/benchmark.py: measure threshold accuracy against a small labeled set
PHASE 6 Performance & Accuracy Tuning ~2–3 days

Accuracy Improvements

  • Add face alignment normalisation: 5-point Umeyama transform (included in InsightFace)
  • Skip faces with det_score < 0.6 (unreliable detections)
  • Use min_quality: 0.4 as default (current 0.5 too aggressive for side profiles)
  • For group photos: run one ANN query per face, not per image
  • Add --calibrate mode in benchmark script: find optimal threshold for a labelled folder pair

Speed Improvements

  • Batch images in chunks of 8 for InsightFace (reduces Python overhead)
  • Use PostgreSQL COPY for bulk face inserts (10× faster than row-by-row)
  • Lazy-load InsightFace model only once at startup (already done, verify)
  • Add --workers N flag for multi-process image scan (use concurrent.futures)
  • Target: <2 sec/image on CPU, <0.3 sec/image with GPU provider

The most impactful single change: replace the identity lookup loop.

# ─── BEFORE: O(n·F) Python loop ──────────────────────────────────────────── for person in existing_persons: person_faces = face_repo.get_by_person(person.person_id) # N DB round-trips for face in person_faces: # deserialise pickle per face similarity = embedding_service.calculate_similarity(new_embedding, face.embedding) if similarity > best_similarity: best_similarity = similarity best_person_id = person.person_id # ─── AFTER: single pgvector ANN query ───────────────────────────────────── # face_repo.find_nearest() runs this SQL: # SELECT cluster_id, person_id, (embedding <=> :qvec) AS dist # FROM faces JOIN clusters USING (cluster_id) # ORDER BY dist LIMIT :k; candidates = face_repo.find_nearest(new_embedding, k=10, threshold=1 - config_threshold) if candidates: best = candidates[0] # already sorted by cosine distance assign_to_person(best.person_id, best.cluster_id) else: create_new_person(new_embedding)

New DB model — swap LargeBinary for pgvector.

from pgvector.sqlalchemy import Vector class FaceDB(Base): __tablename__ = 'faces' face_id = Column(Integer, primary_key=True, autoincrement=True) image_id = Column(Integer, ForeignKey('images.image_id'), nullable=False) cluster_id = Column(Integer, ForeignKey('clusters.cluster_id', ondelete='CASCADE')) embedding = Column(Vector(512), nullable=False) # ← was LargeBinary (pickle) quality = Column(Float, nullable=False) created_at = Column(DateTime, server_default=func.now()) # Index created once in init_db.py: # CREATE INDEX ON faces USING ivfflat (embedding vector_cosine_ops) WITH (lists=100);

InsightFace service — single pipeline, no preprocessing split.

import insightface, cv2, numpy as np class InsightFaceService: def __init__(self, model_pack='buffalo_l', det_size=(640, 640)): self.app = insightface.app.FaceAnalysis( name=model_pack, providers=['CPUExecutionProvider'] # swap to CUDAExecutionProvider for GPU ) self.app.prepare(ctx_id=0, det_size=det_size) def process_image(self, path: str) -> list[dict]: img = cv2.imread(path) faces = self.app.get(img) # detect + align + embed in ONE call results = [] for face in faces: emb = face.embedding / np.linalg.norm(face.embedding) # L2 normalise quality = _quality_score(face, img) results.append({'embedding': emb, 'quality': quality, 'bbox': face.bbox, 'det_score': face.det_score}) return results def _quality_score(face, img) -> float: gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY) x1, y1, x2, y2 = [int(v) for v in face.bbox] crop = gray[y1:y2, x1:x2] blur = min(cv2.Laplacian(crop, cv2.CV_64F).var() / 500.0, 1.0) size = min((x2-x1) * (y2-y1) / 90000, 1.0) return 0.5 * float(face.det_score) + 0.3 * blur + 0.2 * size

Risk Register

Existing pickle data loss
Migration from LargeBinary pickle to pgvector VECTOR(512) requires a one-time conversion script. If it fails mid-way, embeddings are lost.
Write migration script with dry-run mode. Keep LargeBinary column as backup for one release cycle. Use pg_dump before migration.
InsightFace buffalo_l model download
First run downloads ~320 MB from CDN. Requires internet at setup time. Fails silently if network is flaky.
Add scripts/download_models.py as explicit setup step. Document expected model path. Add checksum verification.
pgvector IVFFlat requires training
IVFFlat index needs at least lists × 39 vectors to be useful (~3900 faces for lists=100). Small datasets get no speedup.
Use USING hnsw instead — no training required, good performance at all scales. Switch IVFFlat only for 50k+ faces.
Threshold sensitivity
ArcFace R100 similarity distributions differ slightly between DeepFace and InsightFace preprocessing. Current threshold (0.50) may fragment or over-merge.
Add benchmark script. Default 0.45 is safer for InsightFace cosine. Document tuning guide in README.
Windows path handling
Config uses hardcoded Windows paths (D:\\Photos). Cross-platform users will hit config issues.
Use pathlib.Path throughout (already done). Add platform-agnostic default paths in config template.
ONNX CPU speed on large photo libraries
InsightFace on CPU: ~1–3 sec/face. 5000 photos × avg 2 faces = ~3–8 hours on CPU.
Add GPU provider docs. Add --workers parallelism flag. Progress bar + resume support already via skip_existing.
